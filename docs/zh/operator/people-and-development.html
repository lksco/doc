# 持續發展

這個 ScoreList 系統將會維持於現有版本，除非使用者要求修復或改善。雖機會不大，未來此系統有可能會使用 Google API 簡化程序，令操作員不再需要學習 Google Sheets 的輸入模式。

如你有興趣參與未來 ScoreList 發展，甚或提供另一系統或版本作落實考慮，歡迎與 System Manager 討論。（聯絡資料在上）

# 參與專案

我們歡迎任何人改善 ScoreList，創建自己的軟件體驗。如果你更希望參與專案，可以透過以下方法修改及貢獻至本專案。

要編輯 ScoreList 或 Documentation：

  1. 如果你未有 Github 帳戶 (opens new window)，請先建立一個

對於 ScoreList：

  1. 到 Dev Folder (opens new window) 索取最新版本。建立副本 並編輯該副本。接著，產生文件存取連結。

    怎樣產生 Google Sheets 連結


    按 共用。在「取得連結」下，按 變更為任何知道連結的使用者。接下來你便可以 複製連結。

  2. 前往 外用／開發用 GitHub 專案 之 Issues (opens new window) 發佈你的連結

對於 Documentation：

  1. 在說明文件內，頁面是用 Markdown 格式書寫的，全都位於 /dev/ 資料夾內，後再用 Vuepress (opens new window) 將網頁生成。要編輯內容，你只需編輯相應的 .md 文件。你亦可 使用 Vuepress 預覽頁面 (opens new window)。Fork 外用／開發用 GitHub 專案 (opens new window)，再修改 /dev/ 內的文件
  2. 在 外用／開發用 GitHub 專案 (opens new window) 開 Pull Request，以便更新文件（更多資訊 (opens new window)）
